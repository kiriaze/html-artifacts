    Why: Previously, when two clients modified the same text file concurrently, Acme Sync resolved the conflict by keeping both copies as duplicates (e.g., report.txt and report (conflict).txt). This bloated synced directories and confused users.
    
    
    What: This PR introduces a three-way merge strategy using the common ancestor snapshot stored in the SQLite snapshots table. Text conflicts are merged in-memory; only genuinely unresolvable hunks are flagged and written to the conflict log.
