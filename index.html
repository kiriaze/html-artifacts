html-artifacts GitHub

Interactive HTML docs for your codebase

A Claude Code skill that generates self-contained, interactive HTML instead of Markdown walls — plans, analyses, diagrams, and reports that open directly in your browser. 

git clone https://github.com/kiriaze/html-artifacts ~/.agents/skills/html-artifacts
ln -s ~/.agents/skills/html-artifacts ~/.claude/skills/html-artifacts

Then: Use the html-artifacts skill to generate a [pattern] for [your project].

When to use HTML

Plan, analysis, report, diagram, or explainer?       → HTML  ← this skill
Reviewed as git diff (HANDOFF.md, AGENT_TASKS.md)?   → Markdown
Embedded in GitHub (README, PR body)?                → Markdown
Commit message or chat reply?                        → plain text

How it works

1
Install
Clone the repo and symlink it to ~/.claude/skills/ — no npm, no build step.
2
Invoke
Ask Claude to generate a pattern for your project. The skill loads its 18-pattern framework automatically.
3
Browse
Open the output HTML in your browser — fully self-contained, no server required.

What is this?

html-artifacts is a Claude Code skill. Skills are instructions loaded into Claude's context at session start. When you invoke this skill, Claude follows its 18-pattern framework to generate self-contained HTML files — one for each documentation type — directly into your project's docs/html/ directory.

Read the SKILL.md →

Hook-driven (optional)

Auto-generate on file save

Wire up a Claude Code hook so artifacts regenerate automatically whenever a file under docs/plans/ changes. Add the following to your .claude/settings.json: 

"hooks": {
  "PostToolUse": [{
    "matcher": "Write|Edit",
    "hooks": [{ "type": "command", "command": "node ~/.agents/skills/html-artifacts/hooks/regen.js \"$CLAUDE_TOOL_INPUT_FILE_PATH\"" }]
  }]
}

See hooks/ for the regen script and setup guide.
