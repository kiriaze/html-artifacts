Install

Get started in 30 seconds

Clone the repo into your global agents skills directory. That's it. 

# Clone the skill into your global skills directory
git clone https://github.com/kiriaze/html-artifacts \
  ~/.agents/skills/html-artifacts

# Using Claude? Symlink it/skills directory. (optional but recommended)
ln -s ~/.agents/skills/html-artifacts \
  ~/.claude/skills/html-artifacts

Then ask your model: Use the html-artifacts skill to generate a product-brief for [x].
Alternatively: /html-artifacts generate a product-brief for [x].

When to use HTML

Plan, analysis, report, diagram, or explainer?       → HTML  ← this skill
Reviewed as git diff (HANDOFF.md, AGENT_TASKS.md)?   → Markdown
Embedded in GitHub (README, PR body)?                → Markdown
Commit message or chat reply?                        → plain text

Hook-driven (optional)

Auto-generate on file save

Wire up a Claude Code hook so artifacts regenerate automatically whenever a file under docs/plans/ changes. Add the following to your .claude/settings.json: 

"hooks": {
  "PostToolUse": [{
    "matcher": "Write|Edit",
    "hooks": [{ "type": "command", "command": "node ~/.agents/skills/html-artifacts/hooks/regen.js \"$CLAUDE_TOOL_INPUT_FILE_PATH\"" }]
  }]
}

See hooks/ for the regen script and setup guide.

What is this?

html-artifacts is an Agentic skill that generates self-contained, interactive HTML files instead of Markdown walls for plans, competitor analysis, architecture diagrams, code reviews, reports, and more. When you invoke this skill, your model follows its 18-pattern framework to generate self-contained HTML file(s) — one for each documentation type — directly into your project's docs/html/ directory.

Read the SKILL.md →
html-artifacts

Inspired by Thariq Shihipar's The Unreasonable Effectiveness of HTML — the argument that spatial information, relationships, and structure are better shown than described. An interactive HTML page understood in 5 seconds beats a Markdown wall nobody re-reads.
